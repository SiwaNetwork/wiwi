- **Цель**: Разработка технического проекта, полной конструкторской документации, схемотехники, принципиальных схем
- **Результат**: Утвержденный технический проект, спецификации компонентов, готовность к разработке РКД
- **Себестоимость этапа**: 10,500,000 руб (20% от общего бюджета)
- **Прибыль (20%)**: 2,100,000 руб
- **Бюджет этапа**: 12,600,000 руб

- **Цель**: Разработка рабочей конструкторской документации, программного обеспечения, проекта программы и методик предварительных испытаний
- **Результат**: Полная РКД, ПО, программа и методики предварительных испытаний
- **Себестоимость этапа**: 15,750,000 руб (30% от общего бюджета)

#### Альтернативные схемы оплаты

**Модель 1: Полная предоплата ОКР**
- Заказчик авансом оплачивает всю стоимость разработки (52,500,000 руб)
- Поэтапная оплата этапов разработки не требуется
- Амортизация ОКР в цене изделия = 0 руб

**Гибридная модель: Частичная предоплата ОКР**
- Заказчик авансом оплачивает 50% стоимости разработки (26,250,000 руб)
- Оставшиеся 50% изготовитель инвестирует из собственных средств
- Амортизация ОКР в цене изделия рассчитывается от полной стоимости разработки

**Схема оплаты по этапам (рекомендуемая)**
- Согласно ТЗ применяется поэтапная схема оплаты с авансированием 60% в начале каждого этапа и оплатой 40% после приемки этапа
- Такая схема обеспечивает минимизацию рисков заказчика и контроль качества на каждом этапе

### 8.3 Преимущества поэтапной оплаты для заказчика

Такая схема оплаты обеспечивает следующие преимущества для заказчика:

1. **Минимизация рисков**: Оплата происходит только после достижения конкретных результатов
2. **Контроль качества**: Каждый этап заканчивается демонстрацией достижений
3. **Гибкость**: Возможность корректировки требований между этапами
4. **Финансовая безопасность**: 50% аванс позволяет начать работы, 50% после этапа гарантирует результат
5. **Прозрачность**: Четкие критерии приемки каждого этапа

Эта схема особенно эффективна для сложных проектов с высокой степенью инноваций, где риски технических неудач объективно выше стандартных решений.
